target/network.h file

Network targets allow logs to be sent over a network to a remote log collector or relay.

Contents

  • Reference
    • Enums
    • Functions
    • Defines

There are a number of popular options for the remote end including Splunk, rsyslog, and syslog-ng. Network targets can send messages to these and others, over both IPv4 and IPv6, TCP and UDP.

void stumpless_close_network_target(struct stumpless_target* target)

Closes a network target.

Parameters
target The network target to close.

This function closes the network connection associated with the target before destruction. It also destroys all memory allocated for the target, including the target struct itself.

struct stumpless_target* stumpless_new_network_target(const char* name, enum stumpless_network_protocol network, enum stumpless_transport_protocol transport)

Creates a network target, but does not open it.

Parameters
name The name of the target to open. This is only used for identification of the target.
network The network protocol to use.
transport The transport protocol to use.
Returns The new target if no error is encountered. In the event of an error, NULL is returned and an error code is set appropriately.

A network target can be created with this function if the target needs to be created without initiating a network session. This also allows parameters that are not available in the open function to be set before initiating the session, instead of restarting the session on each change.

The target will stay in a paused state until it is opened using the stumpless_open_target function.

struct stumpless_target* stumpless_open_network_target(const char* name, const char* destination, enum stumpless_network_protocol network, enum stumpless_transport_protocol transport, int options, int default_facility)

Opens a network target.

Parameters
name The name of the target to open. This is only used for identification of the target.
destination The destination to send the messages to. This could be a hostname or an IP address.
network The network protocol to use.
transport The transport protocol to use.
options The options to use for the logging target. This is a bit-wise or of one or more STUMPLESS_OPTION values.
default_facility The facility code to use for entries that do not have a facility value set on them. This should be a STUMPLESS_FACILITY value.
Returns The opened target if no error is encountered. In the event of an error, NULL is returned and an error code is set appropriately.

Network targets allow traditional syslog messages to be sent to a remote server. This function allows the type of network target to be specified by parameters, but if you know that you want a specific type then it is easier to use the simplified functions that specify the type by name, such as the stumpless_open_udp4_target function.

struct stumpless_target* stumpless_set_udp_max_message_size(struct stumpless_target* target, size_t max_msg_size)

Sets the maximum message size of a UDP network target.

Parameters
target The target to be modified.
max_msg_size The new maximum message size for the supplied target.
Returns The modified target if no error is encountered. In the event of an error, NULL is returned and an error code is set appropriately.

Messages that are longer than the maximum size are truncated to this length. If the underlying transport for a network target can support a larger datagram and you need support for longer messages, then you can use this option to increase the size. Likewise, the option can be used to force smaller datagrams if needed, for example in the case of extra overhead in the IP header beyond the typical 20 bytes.

Note that truncations due to the maximum size do not take UTF characters into account. If you have entries using multi-byte characters, then you need to be sure that these truncations do not cause parsing or security problems.

Without calling this function, UDP targets start with a maximum message size set to STUMPLESS_DEFAULT_UDP_MAX_MESSAGE_SIZE.

Define documentation

#define STUMPLESS_DEFAULT_TRANSPORT_PORT

The default transport port for network targets.

This is based largely on tradition, but some applications may expect a different port (for example 1514 and 6514 are sometimes used). If a different port is needed on a target, use stumpless_set_transport_port() to change it.

#define STUMPLESS_DEFAULT_UDP_MAX_MESSAGE_SIZE

The default message size for UDP network targets.

This is set to account for an MTU of 1500 byes, a 20 byte IP header, and an 8 byte datagram header. If you wish to change this value for a particular target, then you must use the stumpless_set_udp_max_message_size() function on the target.
